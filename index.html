3.push、pull及clone

1.push

git push 使用本地的分支来对远程仓库的分支进行更新
$ git push<远程主机名><本地分支名>:<远程分支名>
1）如果省略远程分支名，则表示将本地分支推送与之存在”追踪关系”的远程分支(通常两者同名)，如果该远程分支不存在，则会被新建。
2）如果省略本地分支名，则表示删除指定的远程分支，因为这等同于推送一个空的本地分支到远程分支，这条命令是删除远程master分支。

$ git push origin :master
# 等同于
$ git push origin --delete master

3） 如果当前分支与多个主机存在追踪关系，则可以使用-u选项指定一个默认主机，这样后面就可以不加任何参数使用git push。
$ git push -u origin master
上面命令将本地的master分支推送到origin主机，同时指定origin为默认主机，后面就可以不加任何参数使用git push了。不带任何参数的git push，默认只推送当前分支，这叫做simple方式。

2.pull

git pull 命令是从远程获取最新版本到本地库，用于从另一个存储库或本地分支获取并集成整合。
命令格式：
$ git pull <远程主机><远程分支>:<本地分支>
例如：
$ git pull origin master:my_test
如果省略本地分支，则将自动合并到当前所在分支上。

3.clone

1.什么是web服务器，web服务器有哪些

1）Web服务器一般指网站服务器，是指驻留于因特网上某种类型计算机的程序，可以向浏览器等Web客户端提供文档，也可以放置网站文件，让全世界浏览。
2）web服务器常用的有Apache和Microsoft的Internet信息服务器（Internet Information Services，IIS）。
Web服务器是指驻留于因特网上某种类型计算机的程序。当Web浏览器（客户端）连到服务器上并请求文件时，服务器将处理该请求并将文件反馈到该浏览器上，附带的信息会告诉浏览器如何查看该文件（即文件类型）。服务器使用HTTP（超文本传输协议）与客户机浏览器进行信息交流，这就是人们常把它们称为HTTP服务器的原因。Web服务器不仅能够存储信息，还能在用户通过Web浏览器提供的信息的基础上运行脚本和程序。

2.什么是Web浏览器？Web浏览器软件有哪些

1）浏览器是用来检索、展示以及传递Web信息资源的应用程序。
2）主流的浏览器分为IE、Microsoft Edge、Chrome、Firefox、Safari等几大类

3.安装一个网站软件与安装一个桌面软件有什么不同？

网站软件：

1)需要IE来执行
2)一般以网页的形式的存在
3)可以有很多的客户端来共享。
优点： 1)能够实现最大的资源共享
2)维护与部署起来成本低
缺点： 响应速度比桌面应用程序要低很多，因为要通过Internet去访问远程网页

桌面软件：

1)缺少共享
2)直接双击就可以运行，不需要IE
3)一般以window的窗口形式存在

二.什么是Markdown

1.Markdown的介绍

Markdown 就是一种文档的格式，文件名的末尾是.md，正如我们常用的word文档格式是.doc、.docx，需要对应的软件来打开这一种格式。

2.我们为什么要使用Markdown

Markdown与传统编辑器不同，借助Markdown的语法我们能够更轻易地脱离鼠标操作，而不至于每次想要换个标题就借用鼠标去点击而打断编辑，借助Markdown我们可以大大提高编辑文档的速度

三.Mardown的语法

因为时间有限，只列出部分

  1. Markdown 标题语法
    h1~h5等标题均可借助#来实现，其具体大小依据#的个数来实现

  2. Markdown强调语法
    选中所需要强调的语句，按下Ctrl+B即可实现粗体，按下Ctrl+I即可实现斜体

  3. Markdown的列表语法
    列表分为有序列表及无序列表

  • 有序列表，只需使用 “ 数字. “ + 空格即可实现
  • 无序列表只需使用 “ - “ + 空格即可实现

  1. Markdown的图片语法
    基本形式： “ ! “ + “ [] “ + “ () “
    []中存放的是图片的alt属性
    其中()中即是图片的位置同时也可以包含图片的名字（如 (https://ts1.cn.mm.bing.net/th/id/R-C.7d4958e11e81d434e8e25f58ff5fa5b8?rik=OaHp11zMsCTrng&riu=http%3a%2f%2fi0.hdslb.com%2fbfs%2farticle%2f204b6f842eea4130383f573feddeb2a32bfe2e90.jpg&ehk=k0YlqM2Ccur1zZc1RqDHNheHML75ItK2C2O%2fH%2bsmzPk%3d&risl=&pid=ImgRaw&r=0)）

  2. Markdown的链接语法
    与图片语法类似，链接语法的形式是” [] “ + “ () “
    其中” [] “存放的是链接的名字，而” () “存放的是链接

  3. Markdown的引用语法
    只需使用 “ > “ + 空格 即可实现引用
    例如：

    阿巴阿巴

  4. Markdown的代码语法
    要想在.md文件中显示代码，只需要在代码前含有8个及以上空格或者2个及以上tab键即可实现(注意,在代码开始的上一行还需加一行空行)
    例如：

 <html>
     <head></head>
     <body></body>
 </html>

四. 使用Markdown的感受

与之前使用word的感受不同，使用Markdown我不需要过多的去移动鼠标来调整到我所需要的标题的大小，与此同时，Markdown的列表语法还让我有了清晰的大纲，这是之前使用word都未曾有的体验，但是在使用Markdown写文章的过程中还是遇到了一些麻烦，与word不同，word让我能够随意地对图片之类的位置进行调整，而Markdown这种将内容与布局分开的形式在面对这类问题时缺点也随之体现。总而言之，事物都有两面性，Markdown即使有缺点，但是也不会影响它的优点给我们带来的好处
